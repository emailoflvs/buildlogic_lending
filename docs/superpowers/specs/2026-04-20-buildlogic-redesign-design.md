# BuildLogic — редизайн сайта buildlogic.eu

**Дата:** 2026-04-20
**Статус:** утверждён пользователем, готов к этапу планирования внедрения

---

## 1. Цель

Провести визуальный и содержательный редизайн сайта `buildlogic.eu` (текущая версия — Astro-приложение в `buildlogic_www/`). Задача — подтянуть «технический характер» продукта через чертёжные/строительные элементы в дизайне и заменить российские реалии на универсальные европейские, сохранив текущую структуру страницы.

**Источник вдохновения:** HTML-прототипы из архива `max/example/Buildlogic (1).zip` — Вариант A («Архитектор») и Вариант B («Живой объект»). Из них отобраны конкретные элементы, которые переносятся в боевой сайт.

## 2. Контекст и ограничения

### 2.1 Позиционирование
- **BuildLogic — не ERP-система.** В копирайте, мокапах и документах слово «ERP» упоминать нельзя (ограничение зафиксировано в memory: `buildlogic_not_erp.md`). Альтернативные формулировки: «рабочий контур объекта», «контроль стройки», «система учёта материалов и документов», «construction control system», «stavebný operačný systém».
- **Итоговая формулировка hero:** *«Контроль стройки в одном рабочем контуре»*.
- **Итоговая формулировка Why-манифеста:** текущее «Не большая ERP — это один рабочий контур» переформулировать в «Не десятки модулей — это один рабочий контур, который работает».

### 2.2 Многоязычность
- Равноправные языки: RU / SK / EN.
- Переключатель — pill (три кнопки без флагов), хранение выбора в `localStorage`.
- Все строки — в `src/content/*.json` с тремя полными переводами.

### 2.3 Европейский рынок (EU-адаптация контента)
- Удалить российские реалии: нет М-11, КБ-2в, 1С, БАС.
- Валюта только **EUR** (формат: `128 460 €` с пробел-разделителем, знак после числа).
- Универсальные документы: **Delivery note · Work acceptance · Invoice**.
- Материалы в EU-стандартах: бетон `C25/30`, арматура `⌀12 mm`, кирпич `250×120×65`, цемент `M500`.
- Единицы: `m³ / kg / pc / l`, не tonnes и штуки по-русски.
- Роли: Owner / Director / Site manager / Procurement / Accounting (плюс RU/SK-переводы в словаре).

### 2.4 Технологический контур
- Astro-проект: `src/components/*.astro`, контент в `src/content/*.json`, стили в `src/styles/{tokens,global,fonts}.css`.
- Трогать только фронтенд — без изменений инфраструктуры (`nginx.conf`, `Dockerfile`).

### 3.2 Типографика
- Заголовки: **Fraunces** (opsz 9..144, weight 400/500/600).
- Тело: **Inter Tight** (400/500/600/700).
- Моно-кикеры, цифры KPI: **JetBrains Mono** (400/500).
- H1 — 56-72px, H2 — 40-52px, H3 — 18-22px.

### 3.3 Фон Hero (вариант D — «Гибрид»)
- Миллиметровка 32×32, `stroke: rgba(31,59,52,0.08)`.
- Оси координат `A-F / 1-4` (JetBrains Mono 11px), dashed-линии по крупной сетке.
- Размерная линия сверху: `24 500 mm` между метками-засечками.
- План этажа в правом нижнем углу: прямоугольник с диагональной штриховкой + перегородка + дверная дуга, подпись `PLAN 1:50`.
- Силуэт крана (SVG) в правом верхнем углу, opacity 0.26.
- Силуэт экскаватора в левом нижнем углу, opacity 0.28.

### 3.4 Фон на других секциях
- **Problems**: лёгкая изо-сетка (паттерн из `shared-fx.js`, variant `iso`), opacity ещё ниже (~0.06).
- **Workflow**: миллиметровка (как hero, без доп. элементов).
- **Overview, Pricing, FAQ, Contact**: без сетки, только токены и шрифты.
- **Why**: тёмно-зелёная сплошная + декоративная рамка из 4 угловых меток (L-shaped angle brackets) охрой.

## 5. Анимации и микро-взаимодействия

- **Живая линия** (Hero): SVG-путь с `stroke-dasharray` + `animation: underline-draw 1.8s ease-out forwards`.
- **Sway крана** (Hero background): `rotate(-1.2deg)` ↔ `rotate(1.2deg)`, 6s ease-in-out infinite.
- **Count-up** (все KPI): IntersectionObserver + requestAnimationFrame с cubic ease-out, 1400ms.
- **Авто-шаг** (Hero KPI card): `setInterval` 3200ms, пауза на `onMouseEnter` карточки.
- **Fade-in-up** (появление строк в превью): `translateY(8px)` → `translateY(0)` + opacity 0 → 1, delay по индексу.
- **Module card hover**: `translateY(-2px)` + `box-shadow: 0 40px 60px -30px rgba(31,59,52,0.25)`.
- **Pulse-red** (дефицит): `box-shadow` pulse 1.8s infinite у красных точек.
- Все переходы через `var(--bl-ease)` = `cubic-bezier(.2,.7,.2,1)`.

## 6. Доступность и производительность

- Все интерактивные элементы — `<button>` или `<a>`, не `<div onclick>`.
- `aria-expanded` на FAQ-items.
- `accent-color: var(--bl-green)` на range-input в ROI-калькуляторе (если будет добавлен позже).
- Контраст охры на беже проверяется вручную (WCAG AA для крупного текста).
- Шрифты: только 4 семейства (Fraunces / Inter Tight / JetBrains Mono), в production оставить начертания `400, 500, 600`.
- SVG-фоны встроены inline, не загружаются как `<img>`.

## 7. Границы этой задачи (YAGNI)

**В объёме этого редизайна:**
- Hero, Inside, Workflow, Why — major rewrite.
- Header, RoleStrip, Problems, Overview, Pricing, FAQ, Contact, Footer — tokens + минорные правки.
- Контент `src/content/*.json` — EU-адаптация текста и удаление упоминаний ERP.
- Токены `src/styles/tokens.css` — новая палитра и типографика.

**Не входит:**
- ROI-калькулятор (вынесено в следующий спринт, если понадобится).
- Таблица сравнения с конкурентами (отдельная задача).
- Блог, кейсы, отзывы — отсутствуют в текущем скоупе.
- Инфраструктура (nginx, Docker) — не трогаем.
- A/B-тесты и Tweaks-панель из прототипа — не переносим в прод.

## 8. Риски и mitigation

| Риск | Mitigation |
|------|-----------|
| Астро-компоненты становятся слишком большими (SVG + анимации + mini-UI inline) | Выделять mini-UI как child-компоненты `Hero/KpiCard.astro`, `Inside/BoqCard.astro` и т.п. |
| Переформулировка текста без ERP ломает нюансы смысла | Пересматривать каждый язык отдельно, не машинный перевод |
| Анимации на mobile тормозят | Использовать `@media (prefers-reduced-motion)` для отключения sway/scan/pulse |
| EU-лексика в карточках может не читаться по-русски | Все UI-строки ('Delivery note', 'Work acceptance' и т.д.) — в `i18n.json`, для RU можно оставить английский термин + короткий русский комментарий рядом |
